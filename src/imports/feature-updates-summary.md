Feature 1
Clearer Entry & Immediate Value
User feedback:
“I’m not sure what this does.”
“Am I scanning or taking a photo?”
We refined the entry point to immediately communicate purpose and expectations.
The updated experience:
•	Clearly labels the required photo (“Whole plant photo”)
•	Uses guided framing to show what to capture
•	Establishes visual hierarchy that highlights the primary action
Users now understand what they need to do — and why — from the first screen.
Feature 2
Flexible Photo Capture (Reduced Pressure)
User feedback:
“I felt rushed.”
“I wanted to choose a better photo.”
Real-time scanning created pressure.
We shifted to a more flexible capture model:
•	Clear camera controls
•	Thumbnail previews
•	Ability to select the best image before submission
This change gave users control and reduced anxiety during upload.
The experience now feels intentional instead of reactive.
Feature 3
Structured Diagnostic Questions
User feedback:
“It asked questions, but I didn’t know why.”
“I wasn’t sure how to answer.”
We introduced structured, contextual follow-up questions to support more accurate diagnosis.
The updated flow:
•	Asks about watering frequency
•	Clarifies soil type
•	Guides users through assessing soil moisture
•	Provides descriptive context to reduce guesswork
This reframed the AI from an opaque tool to a collaborative assistant.

Feature 4
Guided Photo & Context Tips
User feedback:
“I don’t know if I took the right kind of photo.”
“What am I supposed to look for?”
We added clearer guidance to improve photo quality and user confidence:
•	Visual capture prompts
•	Contextual instructions
•	Progressive steps that reinforce expectations
This increased both user confidence and diagnostic reliability.

Feature 5
Actionable Treatment Guidance
User feedback:
“Okay… but what do I do now?”
Diagnosis alone wasn’t enough.
We introduced plain-language treatment steps that:
•	Explain the issue clearly
•	Provide specific care recommendations
•	Outline immediate next actions
Users leave the diagnosis screen with clarity and a plan.
Feature 6
Transparent, Confidence-Based Diagnosis
User feedback:
“I don’t know how accurate this is.”
“Why is it saying that?”
The redesigned diagnosis page prioritizes clarity and trust.
Instead of just naming the issue, the updated screen includes:
•	The identified condition (e.g., Overwatered)
•	A visible confidence percentage
•	A “Top Match” label
•	Explanation of how the result was determined
•	A breakdown of symptoms
This transforms the AI output from a black box into an explainable result.
Users can see:
What was detected
Why it was detected
How confident the system is
That transparency directly addressed skepticism uncovered in testing.
Feature 7
Product Recommendations Connected to Diagnosis
User feedback:
“If I need something, just show me.”
“Don’t make me search for it.”
We closed the loop by connecting diagnosis directly to relevant product recommendations.
The updated flow:
•	Curates products specific to the diagnosed issue
•	Connects treatment guidance to purchasable solutions
•	Reduces the need for manual search
The journey now moves seamlessly from:
Problem → Explanation → Treatment → Purchase
Planta evolved from an informational tool into a practical solution engine.
